# 인공지능의 원전 운영 적용 미니 프로젝트
## 2D 중성자 확산 방정식의 U-Net 대리모델 (Surrogate Model)

---

## 1. 문제 정의

### 1.1 배경: 원자로 노심 해석의 계산 비용 문제
원자로 노심(reactor core)의 거동을 예측하려면 중성자 수송/확산 방정식을 수치적으로 풀어야 한다. 산업계에서 가장 널리 쓰이는 단순화 모델은 **2-그룹 중성자 확산 방정식(two-group neutron diffusion equation)** 으로, 노심을 격자(mesh)로 이산화한 뒤 유한차분(FDM)이나 nodal method를 사용해 다중군 플럭스와 유효 증배율 `k_eff` 를 구한다. 이는 고유치(eigenvalue) 문제로 정의되며, 일반적으로 **거듭제곱 반복법(power iteration)** 으로 해결한다.

문제는 다음과 같다.
- 실제 노심 설계·운전 절차(노심장전계획 최적화, 노심 안전 해석, 부하 추종 운전 분석 등)에서는 **수천 ~ 수만 회의 케이스 평가**가 요구된다.
- 결정론적 솔버는 한 케이스당 수 초 ~ 수 분이 걸려, 후보를 모두 평가하는 데 비현실적인 시간이 소요된다.

### 1.2 본 프로젝트의 문제 설정
본 프로젝트는 위 병목을 완화하기 위한 **소형 대리모델(surrogate model) 학습** 을 다룬다. 구체적으로:

- **입력**: 2D 격자(9×9) 위에 정의된 7채널 단면적장(cross-section field)
  - 군별 확산계수 `D₁, D₂`
  - 군별 흡수 단면적 `Σ_a1, Σ_a2`
  - 군별 핵분열 생성 단면적 `νΣ_f1, νΣ_f2`
  - 군간 산란 `Σ_s,2←1` (하향 산란만 고려)
- **출력**:
  - 군별 스칼라 플럭스 맵 `φ₁(x,y), φ₂(x,y)` (2채널 영상)
  - 유효 증배율 `k_eff` (스칼라)

즉, **단면적장 → (플럭스, k_eff)** 라는 회귀(regression) 문제를 정의하고, 결정론적 솔버를 라벨러(labeler)로 사용해 합성 데이터셋을 만든 뒤 U-Net으로 학습한다.

### 1.3 평가 기준
- **플럭스 정확도**: 기준 케이스에 대한 상대 L2 오차 `||φ_pred − φ_true||₂ / ||φ_true||₂`.
- **유효 증배율 정확도**: `|k_pred − k_true|` 와 pcm 단위 오차 (1 pcm = 10⁻⁵).

원자로 물리에서 일반적으로 허용되는 노심해석 코드 검증 기준이 수백 pcm 이내인 점을 고려하여, 본 미니 프로젝트의 목표는 **참고 결정론적 솔버 대비 동일한 수준의 일관성** 을 확보하는 것이다.

## 2. 원자력 분야 관련 연구 소개

### 2.1 결정론적 노심 해석 코드의 흐름
원자로 설계 분야에서는 오랫동안 **격자(lattice) 코드**(예: CASMO, HELIOS, Serpent의 2D 격자 모드)로 단면적을 생산하고, **노심(core) 코드**(예: SIMULATE, PARCS, nTRACER)가 이를 받아 3D 노심 거동을 계산하는 2단계 방법론을 사용해왔다. 이 절차의 마지막 단계가 본 프로젝트가 다루는 **다군 확산 + 거듭제곱 반복** 문제와 본질적으로 같다.

### 2.2 IAEA 2D 벤치마크 문제
연구계에서는 노심 해석 코드의 검증을 위해 **IAEA-2D LWR benchmark** 와 같은 표준 문제를 사용해 왔다. 본 프로젝트의 형상·재료 분포·단면적 표는 IAEA 2D 벤치마크 스타일을 따른 **사분원 노심(quarter-core)** 을 단순화한 9×9 거친(coarse) 격자로 구성된다. 재료 ID는 다음과 같다 (`iaea2d_reference.py:21-33`).

| ID | 의미 |
|----|------|
| 0 | Outside (사분원 영역 밖) |
| 1 | Fuel1 (저농축 연료) |
| 2 | Fuel1 + Rod (제어봉 삽입 위치) |
| 3 | Fuel2 (다른 농축도/조성의 연료) |
| 4 | Reflector (반사체) |

경계조건은 사분원 대칭을 반영해 **상/좌 반사(reflective), 우/하 진공(vacuum)** 으로 설정되어 있다 (`diffusion_solver.py:20-25`).

### 2.3 딥러닝 기반 대리모델 연구 동향
2018년 전후로 **PDE 대리모델로서의 CNN/U-Net** 가 활발히 연구되었다 (예: Zhu & Zabaras (2018) "Bayesian deep convolutional encoder–decoder networks for surrogate modeling and uncertainty quantification"). 원자력 분야에서도 다음과 같은 흐름이 보고되어 왔다.
- **노심 출력분포 예측**: 단면적·연소도 분포로부터 출력분포를 빠르게 추정하는 CNN 대리모델.
- **장전 패턴 최적화(loading pattern optimization)**: 강화학습/유전 알고리즘의 평가 함수로 신경망 대리모델을 사용.
- **불확실성 정량화 (UQ)**: 단면적 perturbation에 대한 출력 응답을 회귀하는 baseline.

본 미니 프로젝트는 이러한 흐름의 가장 단순화된 **2D · 2-group · 정상상태(steady-state)** 버전을 직접 구현해 학습한 사례에 해당한다.

## 3. 데이터 수집 방안 및 현황

### 3.1 데이터 확보 전략: 결정론적 솔버를 통한 합성 데이터 생성
실제 노심 운전 데이터는 비공개·기밀이며, 동일 격자 위에서 단면적장만 변경한 다수 케이스를 얻기 어렵다. 따라서 본 프로젝트는 **결정론적 2-group 확산 솔버를 라벨 생성기**로 사용하는 합성 데이터 전략을 채택한다 (`PLAN.md` Phase 2~3 의 의도와 일치).

본 데이터 생성기는 **구조는 고정하고 연료만 재배치하는** 물리적 사전(prior)을 반영하도록 설계되었다. 단순히 셀마다 독립 균등으로 재료를 뽑으면, (i) 노심 형상을 정의하는 구조물(outside, 반사체)이 학습 분포 내에서 흩어져 비물리적 케이스가 다수 생성되고, (ii) 연료/제어봉 총량이 보존되지 않아 노심 단위로 의미 있는 가설 검정이 어렵다. 따라서 본 파이프라인은 다음 4가지 원칙을 따른다 (`generate_training_data.py`).

1. **기하 고정**: outside (id=0) 11셀과 reflector (id=4) 17셀의 위치는 `MATERIAL_MAP_COARSE` 와 동일하게 모든 샘플에서 유지된다. 노심 형상이 정하는 구조물이므로 무작위화 대상이 아니다.
2. **고정 인벤토리의 순열 재배치**: 연료 위치 53셀에 대해, 기준 맵에서 도출한 **고정 인벤토리** (`Fuel1 × 34, Fuel1+Rod × 4, Fuel2 × 15`) 를 무작위로 셔플해서 재배치한다. 즉 각 샘플은 동일한 총 연료/제어봉 개수를 가진 **순열(permutation)** 이다. 셀별 독립 균등 추출 대비 (a) 각 타입 개수가 보존되어 노심 단위 적분량이 의미를 가지고, (b) 탐색 공간이 훨씬 작아지며, (c) 실제 노심 장전(loading pattern) 최적화 문제 셋업과 정확히 일치한다.
3. **단면적 ±10% 섭동은 유지**: 위 순열의 위에, 양수 단면적 값에 ±10% 균등 곱셈 섭동을 가해 같은 배치 안에서의 단면적 변동성을 모형화한다. 기준값이 0인 항목(OUTSIDE 행, 반사체의 핵분열 단면적 등)은 0으로 고정해 비물리적 케이스를 차단.
4. **사분원 대칭 BC 보존**: 노심 격자는 사분원이고 솔버는 상/좌 reflective + 우/하 vacuum BC를 사용하므로 (`diffusion_solver.py:20-25`), 데이터 생성에서 별도 처리 없이 BC 자체가 사분원 대칭을 가정한다.

### 3.2 데이터 규모 및 held-out 구조 패턴 분할
- **총 샘플 수**: 1,000개 (각 샘플 = 1개 순열 × 1개 단면적 섭동, `generate_training_data.py:177`).
- **고유 구조 패턴**: 1,000 / 1,000 — 다항계수 `53! / (34! · 4! · 15!)` 이 천문학적이라 실측에서도 충돌이 0이다.
- **구조 기반(held-out structural) 분할**: 단순 무작위 분할이 아니라, 생성된 1,000개 패턴을 contiguous chunk 로 700/150/150 으로 잘라 **train / val / test 간 구조 패턴이 절대 겹치지 않도록** 보장한다. 인덱스는 `split_{train,val,test}.npy` 로 저장되며, `main.py:build_dataloaders` 가 이를 우선 사용한다 (없으면 무작위 분할로 fallback).
- **기준 평가 케이스**: 섭동 없는 원본 `MATERIAL_MAP_COARSE` + 원본 단면적으로 별도 저장 (`reference_input.npy` 등). 이는 (a) 학습 분포에 들어가지 않은 구조 패턴이고 (b) 단면적 섭동도 없으므로 **이중으로 분포-외(OOD)** 인 검증용 표본이다.

### 3.3 라벨 솔버 (결정론적 라벨러)
`diffusion_solver.py` 가 제공하는 솔버의 사양:
- 격자 간격 `h = 20 cm`.
- 5점 유한차분(FDM), 셀간 확산계수는 **조화평균(harmonic mean)** 사용해 재료 경계 처리 (`_harmonic_mean`).
- 그룹 1·2 각각에 대해 sparse 선형 시스템 `L · φ = S` 를 SciPy `spsolve` 로 해석.
- 거듭제곱 반복: `tol_keff = 1e-10`, `tol_flux = 1e-8`, `max_iters = 300`.
- 비활성 셀(`material_map == 0`)은 대각 1, 우변 0으로 처리해 시스템에서 강제 분리.

### 3.4 현황 및 한계
- 현재 격자는 9×9 거친 격자(coarse)이며, 실제 IAEA 2D 벤치마크의 핀(pin) 단위 분해능보다 낮다. 이는 **`PLAN.md` 의 Phase 0 (참조 기하 확정)이 아직 partially-only 단계**임을 의미하며, `iaea2d_reference.py` 의 코멘트에서도 "starter representation" 으로 명시되어 있다.
- 1,000 샘플 / ±10% 섭동 / 단일 인벤토리는 학습 가능성 확인용 baseline 규모이며, 분산을 더 넓히려면 (a) 인벤토리 변동 (다른 농축도 mix), (b) 섭동률 확대 (예: 10%, 20%, 30%), (c) 격자 분해능 상향이 후속 단계의 자연스러운 확장이다.

## 4. 방법론 (모델)

### 4.1 모델 선택의 근거: 왜 U-Net인가
- 입력과 출력 모두 **같은 격자 위의 공간장(spatial field)** 이므로, 픽셀 단위 회귀가 가능한 fully-convolutional 구조가 자연스럽다.
- 중성자 플럭스는 **국소 단면적(인접 셀)과 멀리 떨어진 재료 경계(글로벌 정보)** 양쪽에 의존한다. U-Net의 인코더-디코더 + skip connection 구조는 다중 스케일 특징을 모두 보존한 채 픽셀 단위 출력을 만들 수 있다.
- 9×9 격자에 대해 풀스케일 모델은 과적합 위험이 크므로, **TinyUNet** (`main.py:54-111`) 으로 채널 수를 의도적으로 축소했다.

### 4.2 TinyUNet 구조

```
입력 (B, 7, 9, 9)
  └─ enc1 (ConvBlock 7→24)        ─┐ skip
     └─ pool1 + enc2 (24→48)       ─┐ skip
        └─ pool2 + enc3 (48→96)    ─┐ skip
           └─ pool3 + bottleneck (96→128)
              ├─ keff_head: GAP → Linear(128→64) → ReLU → Dropout → Linear(64→1)
              └─ up3 + dec3 (128+96→96)
                 └─ up2 + dec2 (96+48→48)
                    └─ up1 + dec1 (48+24→24)
                       └─ out_flux: 1×1 Conv (24→2)  →  (B, 2, 9, 9)
```

- `ConvBlock` 은 `Conv→BN→ReLU→Dropout2d` 를 두 번 반복하는 표준 블록 (`main.py:36-51`).
- **드롭아웃 p=0.3, weight decay 5e-4** 으로 정규화를 강하게 걸어 1,000 샘플 규모에서의 과적합을 억제.
- **k_eff 헤드는 보틀넥 특징에 글로벌 평균풀링(GAP)을 적용** 한 작은 MLP로, 공간 평균량인 `k_eff` 의 스칼라 특성과 자연스럽게 부합한다.
- 9×9 → 4×4 → 2×2 → 1×1 식으로 홀수 축이 다운샘플될 때 업샘플 결과 크기가 인코더 측과 어긋날 수 있으므로, `F.pad` 로 비대칭 패딩을 추가해 skip concat 시 차원을 정확히 맞춘다 (`main.py:97-107`).

### 4.3 정규화(Normalization)
학습 안정성을 위해 입력·출력 모두 **데이터셋 전역 z-score** 로 정규화한다 (`main.py:121-138`).
- 입력 채널별 평균/표준편차로 표준화 → 채널 간 스케일 차이(예: `D` vs `Σ_a`) 흡수.
- 플럭스 채널별 평균/표준편차로 표준화 → 두 군의 절대 크기 차이 흡수.
- `k_eff` 도 데이터셋 전역 평균·표준편차로 표준화 → 손실항이 다른 항과 동일한 스케일에서 비교되도록 함.
- 추론 시 역정규화하여 물리 단위로 복원 (`infer_reference_case`).

### 4.4 손실 함수
정규화된 공간에서의 멀티태스크 손실로 정의한다.

```
L = MSE(φ̂_norm, φ_norm) + λ · MSE(k̂_norm, k_norm)
```

- 두 항 모두 z-score 정규화 후의 MSE이므로 무차원이며, `λ = keff_weight` 는 두 무차원 항 사이의 단순 비율로 해석된다.
- 기본값 `λ = 10.0` 으로 시작 (`main.py:26`). 이는 한 픽셀짜리 `k_eff` 신호가 81개 픽셀의 플럭스 신호에 비해 그래디언트 기여가 작아지는 것을 보정하기 위함이다. 향후 15.0과 비교 실험할 수 있도록 코멘트로 남겨두었다.

### 4.5 비교 모델: TinyMLP baseline
U-Net의 공간 구조 사전(prior)이 본 문제에 실제로 도움이 되는지 확인하기 위해, **공간 구조를 전혀 가정하지 않는 단순한 완전연결(MLP) baseline** 을 동일한 데이터·정규화·손실 가중치로 별도 학습한다 (`mlp_baseline.py`).

```
입력 (B, 7, 9, 9)
  └─ flatten → (B, 567)
     └─ Linear(567 → 512) → ReLU → Dropout(0.3)
        └─ Linear(512 → 512) → ReLU → Dropout(0.3)
           ├─ flux_head : Linear(512 → 2·9·9)  → reshape (B, 2, 9, 9)
           └─ keff_head : Linear(512 → 1)      → (B,)
```

- 모든 픽셀이 모든 입력 셀과 fully-connected로 연결되어 있어 **국소성/평행이동 등가성(translation equivariance)** 같은 사전을 전혀 사용하지 않는다.
- `forward` 반환 시그니처를 `(flux, keff)` 로 맞춰 두어 U-Net과 동일한 학습 루프·`evaluate`·`infer_reference_case` 를 그대로 재사용한다 (`mlp_baseline.py:54-60`).
- 파라미터 수는 약 **637K** 로, TinyUNet (약 **830K**) 보다 오히려 약간 적은 수준으로 맞춰 "파라미터 부족"이 비교의 교란 변수가 되지 않도록 했다.

## 5. 훈련 및 최적화

### 5.1 학습 설정
| 항목 | 값 | 근거/이유 |
|------|-----|----------|
| Optimizer | Adam | 표준 baseline, 별도 스케줄러 없이도 안정적 |
| Learning rate | 1e-3 | 작은 모델·작은 격자에서 안정 범위 |
| Weight decay | 5e-4 | 명시적 L2 정규화로 1,000 샘플 과적합 억제 |
| Batch size | 32 | 1,000 샘플에서 한 에폭당 ~ 20+ 스텝 확보 |
| Epochs | 3,000 | 모델 용량 대비 충분히 길게, 단 best-val 체크포인트로 보호 |
| Dropout p | 0.3 | 동일 목적 (정규화) |
| Seed | 42 | torch.manual_seed + np.random.seed + DataLoader split generator 모두 고정 |

### 5.2 학습 절차
- 매 에폭마다 train/val 손실을 기록하고, **val loss가 갱신될 때마다 `best_model.pt` 로 저장** (`main.py:220-222`).
- 20 에폭마다 한 줄씩 로그 출력 (긴 학습 시 stdout 폭주 방지).
- 학습 종료 후 best 체크포인트를 다시 로드해 test set 평가 → `(total, flux, keff)` 손실 출력.

### 5.3 평가 및 최종 산출물
학습이 끝나면 **섭동 없는 기준 케이스** 에 대해 다음 평가를 수행한다 (`main.py:284-292`).
1. 기준 입력을 학습 시 사용한 통계로 정규화 → 모델 추론 → 역정규화.
2. 플럭스 상대 L2 오차, `k_eff` 절대 오차를 stdout으로 출력.
3. 시각화 산출물 저장 (`results/` 디렉터리):
   - `training_history.png` (train/val loss, log scale)
   - `reference_comparison.png` (`True φ₁/φ₂` 와 `Pred φ₁/φ₂` 4분할 비교, 상단 제목에 `k_ref`, `k_pred`, `|k_ref - k_pred|` 표기)
   - `best_model.pt` (체크포인트)

### 5.4 최적화 고려사항
- **`keff_weight` 의 민감도**: 두 손실항의 상대 가중치는 결과에 직접적인 영향을 준다. 너무 작으면 `k_eff` 예측이 부정확해지고, 너무 크면 플럭스가 흐려진다. 본 구현은 정규화된 공간에서 비율을 잡았기 때문에 `λ` 의 의미가 단순(`flux MSE 1단위 vs k_eff MSE 1단위`)하다.
- **정규화의 일관성**: 입력 통계는 train+val+test 합쳐서 계산한다. 본 미니 프로젝트에서는 OOD 평가가 아닌 분포-내 회귀 정확도 검증이 목적이므로 허용되지만, 엄격한 OOD 셋업이라면 train-only 통계로 바꿔야 한다.
- **재현성**: 시드 고정 외에도 `DataLoader` 의 `random_split` 에 별도 generator를 주입해 매 실행 동일 분할을 얻는다.

### 5.5 U-Net vs. MLP 비교 결과
동일한 데이터셋(3.1~3.2절의 순열 + ±10% 섭동, held-out 구조 분할)·동일한 하이퍼파라미터(3000 epoch, Adam lr 1e-3, batch 32, dropout 0.3, `λ_keff=10`, seed 42) 로 학습했다. 학습 종료 시 stdout 의 손실값은 **z-score 정규화된 공간에서의 MSE** 라 물리적 의미가 없으므로, 학습 후 train / val / test 각 split 에 대해 **역정규화 후 per-sample 지표** 를 다시 계산했다 (`main.py:report_split_metrics`, `mlp_baseline.py`).

#### 5.5.1 분포-내(in-distribution) 성능 — held-out 구조 패턴 split

| 모델 | split | n | flux rel-L2 (mean / med / max) | k_eff pcm (mean / med / max) |
|------|-------|---|--------------------------------|------------------------------|
| **U-Net** | train | 700 | 0.123 / 0.115 / 0.398 | 366.3 / 297.5 / 1,775.1 |
| | val   | 150 | 0.174 / 0.132 / 1.051 | 674.6 / 505.9 / 3,881.6 |
| | test  | 150 | **0.154** / 0.124 / 0.706 | **659.8** / 433.2 / 3,379.0 |
| **MLP**   | train | 700 | 0.431 / 0.390 / 0.965 | 2,085.2 / 1,813.1 / 12,464.6 |
| | val   | 150 | 0.452 / 0.417 / 0.943 | 2,147.6 / 1,580.0 / 13,643.8 |
| | test  | 150 | **0.426** / 0.394 / 0.906 | **2,205.3** / 1,771.2 / 11,112.9 |

#### 5.5.2 분포-외(out-of-distribution) 성능 — 섭동 없는 원본 IAEA-style 기준 케이스

| 모델 | flux rel-L2 | k_eff pcm | 파라미터 수 |
|------|-------------|-----------|-------------|
| U-Net | **0.265** | **260.1** | 830,371 |
| MLP   | **0.347** | **3,899.3** | 637,091 |

#### 5.5.3 해석 — 데이터 구조를 물리화한 이후의 진단
이전 버전(셀별 독립 균등 추출) 대비 본 버전(구조 고정 + 인벤토리 순열 + held-out 구조 분할) 으로 바꾼 결과, **U-Net test 플럭스 rel-L2 가 0.375 → 0.154 (2.4배 향상), pcm 이 2,287 → 660 (3.5배 향상)** 으로 크게 줄었다. 같은 모델·같은 학습 예산인데 데이터 사전(prior) 이 더 물리적이 되니 학습이 실제로 진행됐다는 뜻이다.

이 새 baseline 에서 in-/out-of-distribution 분해는 다음과 같이 읽힌다.

- **U-Net 의 reference 플럭스 rel-L2 (0.265) 가 test (0.154) 보다 명백히 나쁘다.** 이것이 바로 진짜 OOD 신호다. IAEA-style 원본 배치는 *공간적으로 구조화된* 분포(같은 연료 타입이 군집을 이룸)인 반면 학습 분포는 *무작위 순열* 이라 군집성이 약하다. U-Net 은 후자에는 잘 일반화하지만 전자의 구조적 패턴에는 약간의 페널티를 받는다.
- **U-Net 의 reference k_eff 는 오히려 260 pcm 으로 test 평균 660 보다 좋게** 나오는데, 이건 k_eff 가 노심 적분 스칼라라 공간 오차가 평균에서 상쇄되는 데다 reference 한 표본이 운 좋게 평균쪽으로 떨어졌다고 해석하는 게 안전하다. test pcm 분포의 mean=660 / median=433 / max=3,379 인데, reference 260 은 분포의 좋은 쪽 끝이다.
- **train(0.123) → test(0.154) → reference(0.265)** 라는 단조 증가가 깔끔하게 나타나, *일반화 갭(train→test)* 과 *OOD 갭(test→reference)* 가 처음으로 둘 다 양수로 분리된다. 이전 데이터에서는 분포가 너무 임의적이라 reference 가 오히려 test 보다 좋게 나오는 착시가 있었는데, 그 착시가 사라졌다.
- **MLP 는 train ≈ test (0.43)** 으로 여전히 평탄 — 일반화가 좋아서가 아니라 학습 데이터조차 거의 외우지 못하는 **underfitting** 이다. 공간 구조 prior 없는 fully-connected 모델로는 어떤 데이터 사전을 줘도 본 문제를 풀 수 없다는 부정적 증거가 한 번 더 확인됐다. MLP 의 reference flux 가 test 보다 약간 좋아 보이는 것 (0.347 vs 0.426) 도 절대 오차 자체가 큰 상황에서의 단일 표본 노이즈에 가깝다.

#### 5.5.4 시각적 비교
`results/comparison_unet_vs_mlp.png` 에서:
- **U-Net**: φ₁/φ₂ 의 중앙부 최대 위치와 사선 경계로의 감쇠 패턴을 거의 그대로 재현. 절대 크기에서 약간의 과대평가만 있고 공간 구조가 정확히 보존된다.
- **MLP**: 좌상단 모서리로 출력이 흘러간 형태로, 사분원 노심의 대각선 대칭성이 완전히 깨졌다. 중앙부 최대값 위치를 못 찾는다.

> 주의: U-Net 의 절대 정확도(test 플럭스 rel-L2 0.154, ~660 pcm) 는 향상됐지만 여전히 실제 노심 해석 코드 검증 기준(수십~수백 pcm, 플럭스 수 % 수준)에는 미치지 못한다. 본 비교의 의의는 절대 정확도가 아니라 **(i) 데이터 사전(prior) 의 물리성이 같은 모델의 학습 성공도에 큰 영향을 준다는 점**, **(ii) held-out 구조 분할 + 1점 OOD 표본으로 일반화 갭과 OOD 갭을 분리해 진단할 수 있다는 점** 에 있다.

## 6. 결론

본 미니 프로젝트는 **2D 2-group 중성자 확산 방정식** 의 결정론적 솔버를 **U-Net 기반 대리모델로 대체**하기 위한 end-to-end 파이프라인을 구현하고, 동일 조건에서 **단순 MLP baseline** 과 비교했다. 정리하면:

1. **문제 정의**: IAEA 2D 스타일 사분원 노심(9×9 격자)에서 단면적장 → 플럭스 + `k_eff` 회귀.
2. **데이터**: 기하 고정(outside·reflector 위치 보존) + 고정 인벤토리(Fuel1 × 34, rod × 4, Fuel2 × 15) 의 순열 재배치 + ±10% 단면적 섭동으로 1,000개 합성 케이스 생성. **train/val/test 사이 구조 패턴이 절대 겹치지 않는 held-out 구조 분할** (700/150/150) 사용.
3. **모델**: 3단 인코더-디코더의 **TinyUNet** (보틀넥에 별도 `k_eff` 회귀 헤드), 비교 baseline 으로 **TinyMLP** (2-layer 완전연결).
4. **학습**: 정규화된 멀티태스크 MSE, 3,000 에폭, val-best 체크포인트, 두 모델 모두 동일 하이퍼파라미터.
5. **검증**: train/val/test 각 split 의 per-sample 물리 단위 지표(rel-L2, pcm) 와, 분포-외 reference 1점을 함께 보고.

### 6.1 본 접근의 의의
- 산업적 노심 해석에서 가장 시간이 많이 드는 부분(다중 단면적 시나리오 평가)을, **결정론적 솔버를 라벨러로 쓰는 self-supervised 형태의 합성 데이터 학습**으로 가속할 수 있음을 단순한 형태로 확인.
- 데이터 사전(prior)을 셀별 독립 추출 → **구조 고정 + 인벤토리 순열** 로 바꾸기만 했는데 같은 U-Net 의 test 플럭스 rel-L2 가 **0.375 → 0.154 (2.4배), pcm 이 2,287 → 660 (3.5배)** 향상됐다. 이는 *모델·예산이 아니라 데이터 사전이 학습 성공의 가장 큰 레버* 가 될 수 있음을 단적으로 보여준다. 또한 실제 노심 장전 패턴 최적화 문제 셋업(인벤토리 보존 순열) 과 정확히 일치하는 데이터 분포다.
- 동일 데이터·동일 학습 예산·유사 파라미터 수에서 **U-Net 이 MLP 대비 test 플럭스 rel-L2 2.8배, k_eff pcm 3.3배 더 정확** (0.154 vs 0.426 / 660 vs 2,205). 공간장 회귀에서 convolutional inductive bias 의 가치를 정량적으로 확인.
- **held-out 구조 분할 + 1점 OOD 표본** 로 일반화 갭과 OOD 갭을 처음으로 분리 진단했다 (U-Net: train 0.123 → test 0.154 → reference 0.265 단조 증가).

### 6.2 한계 및 향후 과제
- 절대 오차 수준(U-Net test rel-L2 0.154, ~660 pcm) 은 실제 노심 해석 코드 검증 기준(수십~수백 pcm, 플럭스 수 % 수준) 에는 아직 미치지 못한다. 다음 단계 후보: (1) 데이터 규모 확대 (1k → 10k+), (2) 모델 용량 확대 또는 더 강한 inductive bias (e.g. equivariant CNN), (3) 인벤토리 자체의 변동 도입 (다른 농축도 mix), (4) `keff_weight` 스윕.
- 격자가 9×9 거친 격자에 머물러 있으며, 실제 IAEA 2D 벤치마크 수준의 분해능·재료 분포로의 확장은 `PLAN.md` 의 Phase 0 (참조 데이터 확정) 이 선결 조건.
- 단일 ±10% 섭동 분포·단일 인벤토리에서만 학습되어 있어, 더 넓은 운전 조건(연소도 변화, 제어봉 수 변화 등)으로의 일반화는 별도 검증 필요.
- 3D 노심·핀 단위 분해능·시간 의존 문제로의 확장은 모델 용량·데이터 규모 모두에서 다음 단계의 과제.

### 6.3 실행 방법
```bash

# 2) U-Net 학습 + 기준 케이스 평가 + 그림 저장

# 3) (선택) MLP baseline 학습 + U-Net 과의 비교 산출

```
